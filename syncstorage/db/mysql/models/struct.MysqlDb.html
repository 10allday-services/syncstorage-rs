Fields

inner: Arc<MysqlDbInner>

Synchronous Diesel calls are executed in a tokio ThreadPool to satisfy the Db trait's asynchronous interface.

Arc provides a Clone impl utilized for safely moving to the thread pool but does not provide Send as the underlying db conn. structs are !Sync (Arc requires both for Send). See the Send impl below.

APIs for collection-level locking

Explicitly lock the matching row in the user_collections table. Read locks do SELECT ... LOCK IN SHARE MODE and write locks do SELECT ... FOR UPDATE.

In theory it would be possible to use serializable transactions rather than explicit locking, but our ops team have expressed concerns about the efficiency of that approach at scale.
